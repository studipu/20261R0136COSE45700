## 3. 마스터 템플릿에 anchor·collider 추가 (Day 1-2)

이현지님이 이미 만든 마스터 모델(VRoid 베이스 + 블렌드쉐입 이식)에 **헤어 부위 anchor point**와 **공유 collider**를 추가합니다.

### 3-1) Anchor empty 추가 (Blender)

마스터 .blend를 열고, head bone 자식으로 4개 Empty를 추가:

```python
# scripts/setup_master_anchors.py
import bpy
from mathutils import Vector

ANCHORS = {
    "anchor_front": {"loc": Vector((0.00,  0.05,  0.08)), "type": "PLAIN_AXES"},
    "anchor_side":  {"loc": Vector((0.08,  0.00,  0.00)), "type": "PLAIN_AXES"},  # 좌측, 우측은 mirror
    "anchor_back":  {"loc": Vector((0.00,  0.00, -0.05)), "type": "PLAIN_AXES"},
    "anchor_top":   {"loc": Vector((0.00,  0.10,  0.00)), "type": "PLAIN_AXES"},
}

arm = bpy.data.objects["MasterArmature"]
head_bone = arm.pose.bones["J_Bip_C_Head"]
head_world = arm.matrix_world @ head_bone.head

for name, cfg in ANCHORS.items():
    bpy.ops.object.empty_add(type=cfg["type"], location=head_world + cfg["loc"])
    empty = bpy.context.object
    empty.name = name
    empty.parent = arm
    empty.parent_type = 'BONE'
    empty.parent_bone = "J_Bip_C_Head"
    empty.empty_display_size = 0.02
```

## 4. Front 1종 PoC — 처음부터 끝까지 (Day 3-5)

이 단계는 자동화 스크립트의 **정답 데이터**를 만들기 위한 수동 작업입니다. 한 번 완전히 돌려본 뒤 자동화에 그대로 옮깁니다.

### 4-1) VRoid Studio에서 헤어 디자인 (~30분)

1. VRoid Studio 실행 → "Create New" → "Female (F)" 베이스
2. 좌측 메뉴 **Hair** 탭 → **Add Procedural Hair Guides**
3. 짧은 일자 앞머리(`front_001_short_straight_fringe`) 디자인:
   - **Hair count:** 30~40
   - **Length:** 8cm
   - **Cross-section:** 0.7mm × 4mm (납작한 카드)
   - **Bend:** 약간 안쪽으로
4. **Material** → 무채색 짙은 갈색(#3a2820)으로 통일 — 색상은 우리 시스템에서 동적으로 변경
5. 다른 부위(Side/Back/Top)는 **모두 비활성화 또는 삭제** — 이번 export는 Front 한 종류만

### 4-2) VRM Export (~5분)

1. **Export → VRM 1.0** 선택
2. Export 옵션:
   - ✅ Include hair
   - ✅ Include rigging (SpringBone)
   - ❌ Include face shapes (블렌드쉐입은 우리 마스터가 갖고 있음)
   - ❌ Include clothing
3. 저장 위치: `hair-library/raw/front_001.vrm`

### 4-3) Blender에서 후처리 (~20분)

```python
# scripts/build_hair_part.py (수동 실행 부분)
import bpy

# 1. VRoid에서 export한 vrm 열기
bpy.ops.import_scene.vrm(filepath="hair-library/raw/front_001.vrm")
vroid_avatar = bpy.context.selected_objects[0]

# 2. 헤어 메시만 분리
#    VRoid 모델은 보통 'Hair001' 같은 이름의 메시들로 헤어가 구성됨
hair_meshes = [obj for obj in vroid_avatar.children
               if obj.type == 'MESH' and 'Hair' in obj.name]

# 3. 모든 헤어 메시를 하나로 join
bpy.ops.object.select_all(action='DESELECT')
for m in hair_meshes:
    m.select_set(True)
bpy.context.view_layer.objects.active = hair_meshes[0]
bpy.ops.object.join()
hair = bpy.context.object
hair.name = "front_001"

# 4. VRoid armature에서 분리 (parent clear, keep transform)
bpy.ops.object.parent_clear(type='CLEAR_KEEP_TRANSFORM')

# 5. 우리 마스터 템플릿 append
bpy.ops.wm.append(filepath="master/master_template.blend/Object/MasterArmature", ...)
master_arm = bpy.data.objects["MasterArmature"]

# 6. 우리 마스터의 anchor_front 위치로 이동
anchor = bpy.data.objects["anchor_front"]
hair.location = anchor.matrix_world.translation
hair.parent = master_arm
hair.parent_type = 'BONE'
hair.parent_bone = "J_Bip_C_Head"

# 7. Bone retargeting — VRoid 헤어의 swing bone들을 마스터 head bone 자식으로 재구성
retarget_swing_bones(vroid_avatar.find_armature(), master_arm, hair)

# 8. SpringBone 파라미터 우리 표준으로 재설정
apply_our_spring_params(master_arm, hair, part_type="front")

# 9. 단독 GLB export
bpy.ops.object.select_all(action='DESELECT')
hair.select_set(True)
master_arm.select_set(True)
bpy.ops.export_scene.gltf(
    filepath="output/front/front_001/mesh.glb",
    use_selection=True,
    export_animations=False,
    export_extras=True,  # SpringBone 메타데이터
)
```

### 4-4) 검증 (~10분)

1. **Three.js 뷰어 (백승우님 영역)에서 확인:**
   - 마스터 + front_001 attach → 머리 회전 시 헤어가 자연스럽게 따라오는지
   - SpringBone 흔들림이 너무 강하거나 약하지 않은지
   - 다른 부위(side/back) 자리에 z-fighting 없는지
2. **VSeeFace에서 확인:**
   - export된 단독 GLB가 아니라 마스터 + 헤어 합친 VRM을 열어봄
   - 웹캠 머리 흔들 때 헤어 흔들림 자연스러움

## 5. 자동화 스크립트 — 35종 일괄 처리

PoC에서 검증된 흐름을 Blender MCP `execute_blender_code`로 일괄 실행:

```python
# scripts/build_all.py — Blender MCP에 통째로 던지는 스크립트
import bpy, json, os, sys
from mathutils import Vector

# ─────── 메타데이터: 35종 정의 ───────
LIBRARY = [
    # FRONT 12종
    {"id": "front_001", "part_type": "front", "vroid_file": "raw/front_001.vrm",
     "display_name_kr": "짧은 일자 앞머리",
     "tags": {"length": "short",  "style": "straight", "bangs": "fringe"}},
    {"id": "front_002", "part_type": "front", "vroid_file": "raw/front_002.vrm",
     "display_name_kr": "짧은 사이드 스윕",
     "tags": {"length": "short",  "style": "straight", "bangs": "swept"}},
    # ... front_003~012
    
    # SIDE 8종
    {"id": "side_001", "part_type": "side", "vroid_file": "raw/side_001.vrm",
     "display_name_kr": "짧은 옆머리", "tags": {"length": "short", "style": "straight"}},
    # ...
    
    # BACK 10종
    {"id": "back_001", "part_type": "back", "vroid_file": "raw/back_001.vrm",
     "display_name_kr": "짧은 뒷머리", "tags": {"length": "short", "style": "straight"}},
    # ...
    
    # TOP 5종
    {"id": "top_001", "part_type": "top", "vroid_file": "raw/top_001.vrm",
     "display_name_kr": "단일 아호게", "tags": {"length": "short", "style": "straight"}},
    # ...
]

# ─────── PART_ANCHORS, SPRING_RULES 설정 ───────
PART_ANCHORS = {
    "front": ("anchor_front", (0.00,  0.05,  0.08)),
    "side":  ("anchor_side",  (0.08,  0.00,  0.00)),
    "back":  ("anchor_back",  (0.00,  0.00, -0.05)),
    "top":   ("anchor_top",   (0.00,  0.10,  0.00)),
}

SPRING_BASE = {
    "front": {"stiffness": 0.85, "drag": 0.50, "gravity": 0.30, "hit_radius": 0.015},
    "side":  {"stiffness": 0.75, "drag": 0.45, "gravity": 0.30, "hit_radius": 0.018},
    "back":  {"stiffness": 0.55, "drag": 0.35, "gravity": 0.40, "hit_radius": 0.020},
    "top":   {"stiffness": 0.90, "drag": 0.40, "gravity": 0.20, "hit_radius": 0.012},
}

def auto_spring(part_type, mesh_length_m):
    base = dict(SPRING_BASE[part_type])
    if mesh_length_m > 0.30:
        base["stiffness"] *= 0.7
    if mesh_length_m > 0.50:
        base["stiffness"] *= 0.7
    return base


# ─────── 1종 처리 함수 ───────
def build_one(meta, master_blend, out_root):
    bpy.ops.wm.read_factory_settings(use_empty=True)
    
    # 마스터 + anchor 로드
    bpy.ops.wm.append(filepath=f"{master_blend}/Object/MasterArmature", ...)
    arm = bpy.data.objects["MasterArmature"]
    
    # VRoid VRM import
    bpy.ops.import_scene.vrm(filepath=meta["vroid_file"])
    vroid_root = bpy.context.selected_objects[0]
    
    # 헤어 메시 분리 + join
    hair_meshes = [o for o in vroid_root.children_recursive
                   if o.type == 'MESH' and 'Hair' in o.name]
    if not hair_meshes:
        raise RuntimeError(f"No hair mesh in {meta['vroid_file']}")
    
    bpy.ops.object.select_all(action='DESELECT')
    for m in hair_meshes: m.select_set(True)
    bpy.context.view_layer.objects.active = hair_meshes[0]
    bpy.ops.object.join()
    hair = bpy.context.object
    hair.name = meta["id"]
    bpy.ops.object.parent_clear(type='CLEAR_KEEP_TRANSFORM')
    
    # anchor 정렬
    anchor_name, _ = PART_ANCHORS[meta["part_type"]]
    anchor = bpy.data.objects[anchor_name]
    hair.location = anchor.matrix_world.translation
    hair.parent = arm
    hair.parent_type = 'BONE'
    hair.parent_bone = "J_Bip_C_Head"
    
    # SpringBone 재설정
    mesh_len = max(hair.dimensions)  # 가장 긴 축
    sp = auto_spring(meta["part_type"], mesh_len)
    chains = setup_swing_bones(arm, hair, meta["part_type"], **sp)
    
    # MToon 머티리얼
    convert_to_mtoon(hair)
    
    # output 디렉토리
    out_dir = os.path.join(out_root, meta["part_type"], meta["id"])
    os.makedirs(out_dir, exist_ok=True)
    
    # GLB export
    bpy.ops.object.select_all(action='DESELECT')
    hair.select_set(True); arm.select_set(True)
    bpy.ops.export_scene.gltf(
        filepath=f"{out_dir}/mesh.glb",
        use_selection=True, export_extras=True,
    )
    
    # 썸네일
    render_thumb(arm, hair, f"{out_dir}/thumb_front.png", yaw=0)
    render_thumb(arm, hair, f"{out_dir}/thumb_side.png",  yaw=90)
    
    # DINOv2 임베딩 (별도 Python 프로세스)
    os.system(f"python scripts/extract_dinov2_emb.py "
              f"--in {out_dir}/thumb_front.png --out {out_dir}/embedding.npy")
    
    # meta.json
    full_meta = {
        **meta,
        "geometry": {"vertex_count": len(hair.data.vertices)},
        "rigging": {"swing_bone_chains": chains, "spring_params": sp,
                    "uses_shared_colliders": ["head_collider", "neck_collider"]},
        "material": {"default_color": "#3a2820", "shader": "MToon",
                     "supports_color_change": True},
        "license": {"source": "self_made_vroid", "author": "이현지",
                    "commercial_use": True},
    }
    with open(f"{out_dir}/meta.json", "w") as f:
        json.dump(full_meta, f, ensure_ascii=False, indent=2)
    
    print(f"✓ {meta['id']} done")


# ─────── 일괄 실행 ───────
if __name__ == "__main__":
    for meta in LIBRARY:
        try:
            build_one(meta, "master/master_template.blend", "output/")
        except Exception as e:
            print(f"✗ {meta['id']} FAILED: {e}")
            with open("build_errors.log", "a") as f:
                f.write(f"{meta['id']}\t{e}\n")
```

## 8. 품질 기준 체크리스트

### 8-1) 메시
- [ ] Vertex count ≤ 5,000 (모바일 호환)
- [ ] Triangle count ≤ 10,000
- [ ] UV가 0~1 범위로 unwrap됨
- [ ] Normal 방향 일관성 (안→밖)
- [ ] Face orientation 정상 (Blender → Overlays → Face Orientation 파란색)
- [ ] Non-manifold edge 없음
- [ ] Texture 1024×1024 (mobile은 512 mipmap)

### 8-2) 본·리깅
- [ ] Swing bone chain 좌우 분리 (front/side는 L/R 각각)
- [ ] Chain 당 본 3~5개
- [ ] Skin weight: 한 vertex당 1~2개 본만 영향
- [ ] Armature 이름이 마스터와 동일

- [ ] `stiffness`, `dragForce`, `gravityPower`, `hitRadius` 4개 모두 설정
- [ ] Collider는 마스터 공유 collider만 참조 (자체 collider 없음)

- [ ] Alpha cutout 또는 transparent 모드 명시

## 9. DB 등록 + 검증 (W4)

### 9-1) 35종 일괄 DB seed

```python
# scripts/db_seed.py
import os, json, numpy as np, psycopg2
from psycopg2.extras import Json

conn = psycopg2.connect(...)
cur = conn.cursor()

for part_type in ["front", "side", "back", "top"]:
    for entry_id in os.listdir(f"output/{part_type}/"):
        entry_dir = f"output/{part_type}/{entry_id}"
        with open(f"{entry_dir}/meta.json") as f:
            meta = json.load(f)
        emb = np.load(f"{entry_dir}/embedding.npy").tolist()
        
        cur.execute("""
            INSERT INTO hair_parts
              (id, part_type, display_name_kr, mesh_path, thumb_front, thumb_side,
               embedding, length_class, style_class, volume_class, bangs_type, meta)
            VALUES (%s,%s,%s,%s,%s,%s,%s::vector,%s,%s,%s,%s,%s)
            ON CONFLICT (id) DO UPDATE SET
              embedding = EXCLUDED.embedding,
              meta = EXCLUDED.meta
        """, (
            meta["id"], part_type, meta["display_name_kr"],
            f"s3://hair-library/{part_type}/{entry_id}/mesh.glb",
            f"s3://hair-library/{part_type}/{entry_id}/thumb_front.png",
            f"s3://hair-library/{part_type}/{entry_id}/thumb_side.png",
            emb,
            meta["tags"]["length"], meta["tags"]["style"],
            meta["tags"].get("volume", "normal"),
            meta["tags"].get("bangs"),
            Json(meta),
        ))

conn.commit()
print("DB seed complete")
```

## 10. 트러블슈팅

| 증상 | 원인 / 해결 |
|---|---|
| **VRoid에서 export한 헤어가 Blender에서 안 보임** | VRM Add-on 활성화 안 됨 → Edit > Preferences > Add-ons에서 "VRM format" 체크 |
| **헤어 메시가 마스터에 너무 작거나 큼** | VRoid 베이스와 우리 마스터 스케일 차이 → import 직후 `bpy.ops.transform.resize`로 보정 (보통 ×0.95 ~ ×1.05) |
| **Anchor 위치에 안 붙고 발 아래 떨어짐** | parent_bone 설정 누락 → `parent_type = 'BONE'` + `parent_bone = "J_Bip_C_Head"` 필수 |
| **SpringBone이 안 흔들림** | Joint root가 head bone 자식이 아님 → swing chain의 root_bone parent를 J_Bip_C_Head로 설정 |
| **Three.js에서 헤어가 머리 안쪽으로 박힘** | Z-axis 반대 → import 후 `hair.rotation_euler.z = pi` |
| **VSeeFace에서 헤어가 사라짐** | MToon 알파 설정 → cutout 모드로 변경, alphaTestThreshold = 0.5 |
| **DINOv2 임베딩이 모두 비슷한 값** | 썸네일이 너무 어둡거나 헤어가 안 보임 → 카메라 각도/조명 재조정 |
