# 부위 분리형 헤어 프리셋 라이브러리 설계

> VRoid Studio 모델을 차용해 부위별로 분리하고, VARCO 입력에서 자동 매칭으로 조합하는 시스템
> 작성일: 2026-04-29

---

## 0. 결정 한 줄

> 부위 분리는 **에셋 비용은 1.3배지만 사용자 만족도 결과 가짓수가 100배 이상**이 되는 최고 가성비 결정. 단, "조합 폭발"을 잘 다루기 위해 **부위 4개·총 35종**으로 시작하고 **호환성 점수**로 자연스러운 조합만 추천한다.

## 1. 왜 부위 분리가 답인가 — 숫자로

| 방식 | 에셋 수 | 결과 가짓수 | 이현지님 작업 시간 |
|---|---|---|---|
| 단일 헤어 30종 | 30 | 30 | ~30시간 |
| **부위 분리 (Front 12 · Side 8 · Back 10 · Top 5)** | **35** | **12 × 8 × 10 × 5 = 4,800** | ~40시간 (Blender MCP로 ~5시간) |

**부수 효과:**
- 자동 매칭 정확도가 부위별로 좁아져 ↑ (앞머리 임베딩은 앞머리끼리만 비교)
- 사용자가 "앞머리만 바꾸기" 같은 미세 편집 가능 — UX 가치 큼
- VRoid 무료 에셋의 부위별 분리/개별 활용이 라이선스상 일반적으로 허용

## 2. 부위 분류 — 4개로 단순화 (VRoid의 6개 → 4개)

VRoid는 6부위(Front/Side/Back/Extension/Ahoge/Extra)지만, MVP에는 과함. 매칭 정확도와 제작 비용을 고려해 **4부위**로 시작:

| # | 부위 | 우리 코드 | 시각적 영향력 | 매칭 난이도 (정면 사진) | MVP 종 수 |
|---|---|---|---|---|---|
| 1 | **Front (앞머리)** | `front` | ★★★★★ — 가장 큼 | 쉬움 (이마 위 명확) | **12** |
| 2 | **Side (옆머리)** | `side` | ★★★★ | 쉬움 (얼굴 좌우) | **8** |
| 3 | **Back (뒷머리)** | `back` | ★★★★ — 길이 결정 | 어려움 (정면 안 보임) | **10** |
| 4 | **Top/Ahoge (정수리·삐친머리)** | `top` | ★★ — 옵션, 캐주얼 | 중 (수직 돌출 검출) | **5** |

**Phase 2에서 추가 후보:** Extension(긴 헤어 보강), Hair Accessories(헤어핀·헤어밴드).

**왜 6→4로 줄였나:**
- Extension은 Back과 시각적 분리 어려움 (정면 1장에서)
- Extra는 거의 사용 안 됨 (VRoid 사용자 통계상 18% 미만)
- Ahoge와 Top을 합치는 게 매칭 친화적

## 4. 본 구조 — 한 armature에 부위별 chain

마스터 템플릿이 VRM 표준 본(Hips, Spine, Neck, Head)을 가짐. 각 부위 헤어가 attach될 때 head bone 아래에 swing bone chain을 추가:

```
J_Bip_C_Head
├── hair_front_l_root  ←  attach point (anchor_point 기준 자동 정렬)
│   └── hair_front_l_01 → hair_front_l_02 → hair_front_l_03  (spring)
├── hair_front_r_root
│   └── hair_front_r_01 → ... 
├── hair_side_l_root
│   └── hair_side_l_01 → ...
├── hair_side_r_root
├── hair_back_root
│   └── hair_back_01 → hair_back_02 → ... → hair_back_05  (가장 긴 chain)
└── hair_top_root
    └── hair_top_01  (아호게는 보통 짧음)
```

**Collider는 마스터 템플릿에 사전 정의:**
- `head_collider` (구체, head bone 기준 r=0.10m)
- `neck_collider` (캡슐, neck~chest)
- `shoulder_l_collider`, `shoulder_r_collider`

각 헤어 부위는 `uses_shared_colliders`로 어떤 collider와 충돌할지 선언만. 따라서 **부위별로 collider를 따로 만들 필요 없음** → VRM SpringBone JSON 단순화.

## 5. 자동 매칭 알고리즘 — 부위별 분리 매칭

### 5-1) 전체 파이프라인

```
사용자 정면 사진 (1장)
   ↓
[A] BiSeNet → hair_mask (binary)
[B] MediaPipe → 478 face landmarks
   ↓
[C] split_hair_regions(mask, landmarks)
   ↓
{ 'front': crop, 'side': crop, 'back': crop|None, 'top': crop|None }
   ↓
[D] 각 부위 → DINOv2 768-dim 임베딩
   ↓
[E] pgvector top-3 매칭 (부위별 라이브러리 안에서)
   ↓
[F] 3×3×3×3 = 81개 조합 → compatibility_score 계산
   ↓
[G] 상위 3개 조합을 사용자에게 미리보기로 제시
```

### 5-2) 부위별 영역 분리 (코드)

```python
import numpy as np
from PIL import Image

def split_hair_regions(image_np, hair_mask, landmarks):
    """
    image_np: HxWx3 RGB
    hair_mask: HxW binary
    landmarks: MediaPipe 478 landmarks (이미지 좌표)
    
    returns: {part_type: cropped PIL or None}
    """
    h, w = image_np.shape[:2]

    # 주요 랜드마크 (MediaPipe FaceMesh 인덱스)
    forehead_y = int(landmarks[10][1])      # 이마 위
    nose_y     = int(landmarks[1][1])
    chin_y     = int(landmarks[152][1])
    left_face  = int(landmarks[234][0])     # 얼굴 좌측
    right_face = int(landmarks[454][0])     # 얼굴 우측
    eyebrow_y  = int(landmarks[107][1])     # 눈썹

    regions = {}

    # ─── FRONT (앞머리) ───
    # 이마/눈썹 위 + 얼굴 폭 안쪽 (약간 양쪽으로 여유)
    front = hair_mask.copy()
    front[eyebrow_y:, :] = 0
    margin = int((right_face - left_face) * 0.2)
    front[:, :max(0, left_face - margin)] = 0
    front[:, min(w, right_face + margin):] = 0
    regions['front'] = _crop_or_none(image_np, front, min_pixels=300)

    # ─── SIDE (옆머리) ───
    # 얼굴 좌우 외곽 + 코~턱 사이 높이
    side = hair_mask.copy()
    side[:nose_y, :] = 0
    side[chin_y:, :] = 0
    side[:, left_face:right_face] = 0
    regions['side'] = _crop_or_none(image_np, side, min_pixels=300)

    # ─── BACK (뒷머리) ───
    # 어깨까지 내려오는 부분 → 정면에서 일부 보임
    # 턱 아래 + 어깨 위 + 얼굴 폭 밖에 있는 hair_mask 영역
    back = hair_mask.copy()
    back[:chin_y, :] = 0
    shoulder_y = min(h - 1, chin_y + int((chin_y - nose_y) * 1.5))
    back[shoulder_y:, :] = 0
    back[:, left_face:right_face] = 0  # 얼굴 가운데(목)는 제외
    regions['back'] = _crop_or_none(image_np, back, min_pixels=200)
    # 정면 사진에서 안 보이면 None → 매칭 시 length 추정으로 보완

    # ─── TOP (정수리/아호게) ───
    head_top_y = _find_head_top(hair_mask)
    top = hair_mask.copy()
    top[head_top_y + 25:, :] = 0
    if _has_protruding_strands(top):
        regions['top'] = _crop_or_none(image_np, top, min_pixels=80)
    else:
        regions['top'] = None  # 아호게 없음

    return regions


def _crop_or_none(img, mask, min_pixels):
    if mask.sum() < min_pixels:
        return None
    ys, xs = np.where(mask > 0)
    bbox = (xs.min(), ys.min(), xs.max(), ys.max())
    cropped = img[bbox[1]:bbox[3], bbox[0]:bbox[2]]
    return Image.fromarray(cropped)


def _find_head_top(mask):
    ys = np.where(mask.any(axis=1))[0]
    return ys[0] if len(ys) else 0


def _has_protruding_strands(top_mask):
    # 머리 위로 튀어나온 연속 영역이 1개 이상이면 아호게로 간주
    from scipy import ndimage
    labels, n = ndimage.label(top_mask)
    return n >= 1 and top_mask.sum() > 100
```

### 5-3) 임베딩 + pgvector 매칭

```python
import torch
from transformers import AutoModel, AutoImageProcessor

dino = AutoModel.from_pretrained("facebook/dinov2-base").eval()
proc = AutoImageProcessor.from_pretrained("facebook/dinov2-base")

@torch.no_grad()
def embed(crop_pil):
    inputs = proc(images=crop_pil, return_tensors="pt")
    out = dino(**inputs).last_hidden_state.mean(dim=1)
    return out[0].cpu().numpy()  # (768,)


def topk_in_part(part_type, emb, k=3):
    rows = db.fetchall("""
        SELECT id, length_class, style_class, volume_class, bangs_type,
               1 - (embedding <=> %s::vector) AS sim
        FROM hair_parts
        WHERE part_type = %s
        ORDER BY embedding <=> %s::vector
        LIMIT %s
    """, (emb.tolist(), part_type, emb.tolist(), k))
    return rows
```

### 5-4) 호환성 점수 + Top-3 조합

```python
LENGTH_RANK = {"short": 0, "medium": 1, "long": 2, "extra_long": 3}

def compatibility_score(combo):
    """combo = {'front': part, 'side': part, 'back': part|None, 'top': part|None}"""
    parts = [p for p in combo.values() if p is not None]

    # 1) 길이 일관성 — extra_long + short 같이 쓰면 부자연스러움
    lengths = [LENGTH_RANK[p['length_class']] for p in parts if p.get('length_class')]
    length_var = (max(lengths) - min(lengths)) if len(lengths) > 1 else 0
    length_score = 1.0 - min(length_var / 3.0, 1.0)

    # 2) 스타일 일관성 — straight/wavy/curly가 한 사람 머리에 다 섞이면 어색
    styles = [p['style_class'] for p in parts]
    most = max(set(styles), key=styles.count)
    style_score = styles.count(most) / len(styles)

    # 3) 볼륨 일관성
    vols = [p['volume_class'] for p in parts]
    vol_score = vols.count(max(set(vols), key=vols.count)) / len(vols)

    # 4) DINOv2 유사도 평균 (사용자 사진과 부위별 닮음 정도)
    sim_score = sum(p['sim'] for p in parts) / len(parts)

    return 0.30*length_score + 0.20*style_score + 0.10*vol_score + 0.40*sim_score


def best_combos(candidates):
    """candidates = {part_type: [top3 rows]}"""
    from itertools import product
    combos = []
    for f, s, b, t in product(
        candidates['front'],
        candidates['side'],
        candidates['back'] if candidates['back'] else [None],
        candidates['top']  if candidates['top']  else [None],
    ):
        combo = {'front': f, 'side': s, 'back': b, 'top': t}
        combos.append((combo, compatibility_score(combo)))
    combos.sort(key=lambda x: -x[1])
    return combos[:3]
```

# 메시 길이 → SpringBone 파라미터 룰 (튜닝 매트릭스)
def auto_spring_params(part_type, mesh_length_m):
    base = {
        "front": {"stiffness": 0.85, "drag": 0.50, "gravity": 0.30},
        "side":  {"stiffness": 0.75, "drag": 0.45, "gravity": 0.30},
        "back":  {"stiffness": 0.55, "drag": 0.35, "gravity": 0.40},
        "top":   {"stiffness": 0.90, "drag": 0.40, "gravity": 0.20},
    }[part_type]
    # 긴 메시일수록 stiffness 낮춤 (자연스러운 흔들림)
    if mesh_length_m > 0.30:
        base["stiffness"] *= 0.7
    return base

    # 1. 마스터 템플릿 로드
    bpy.ops.wm.append(filepath=f"{master_template}/Object/MasterAvatar", ...)

    bpy.ops.import_scene.gltf(filepath=raw_glb)
    hair = bpy.context.selected_objects[0]
    hair.name = meta["id"]

        **meta,

        "geometry": {"vertex_count": len(hair.data.vertices), "anchor_point": anchor["offset"]},
    }
    with open(f"{out_dir}/{part_type}/{meta['id']}/meta.json", "w") as f:

## 7. UX — 추천 + 직접 조합 하이브리드

### 7-1) 메인 흐름 (Three.js 웹뷰어, 백승우님 영역)

1. **자동 추천 단계:** 사진 업로드 후 5초 안에 Top-3 조합 미리보기 카드 3장
2. **선택:** 사용자가 카드 1장 클릭 → 해당 조합이 마스터 템플릿에 즉시 적용
3. **세부 조정:** "헤어 다듬기" 버튼 → 부위별 picker 열림
4. **직접 조합 모드:** 부위 4개 각각 그리드(12 + 8 + 10 + 5 종)에서 선택 → 즉시 반영
5. **색상 변경:** 통일 색상 슬라이더 (HSV) — Phase 2에서 부위별 색상 분리

### 7-2) Three.js 부위 attach/detach (코드 스케치)

```typescript
// 각 부위 헤어 GLB을 미리 로드해두고 캐시
const partCache = new Map<string, GLTF>();

async function applyHairCombo(combo: HairCombo) {
  // 기존 헤어 부위 모두 제거
  HAIR_PART_TYPES.forEach(t => removeHairPart(t));
  
  // 각 부위 attach
  for (const part of HAIR_PART_TYPES) {
    const id = combo[part];
    if (!id) continue;
    
    let gltf = partCache.get(id);
    if (!gltf) {
      gltf = await gltfLoader.loadAsync(`/preset/${part}/${id}/mesh.glb`);
      partCache.set(id, gltf);
    }
    
    const hairMesh = gltf.scene.clone();
    const headBone = avatarVRM.humanoid.getNormalizedBoneNode('head');
    headBone!.add(hairMesh);
    
    // SpringBone joint 등록
    avatarVRM.springBoneManager?.joints.push(
      ...buildSpringJoints(hairMesh, presetMeta[id].rigging)
    );
  }
  
  avatarVRM.springBoneManager?.reset();
}
```

각 부위 GLB은 작아서(20~80KB) 한 번만 로드 후 캐시. 조합 변경은 < 100ms.

## 8. 호환성·자연스러움 보장 — 룰 + 데이터

### 8-1) 룰 기반 (MVP)
- `compatibility_score`로 자동 추천 시 자연스러운 조합만 상위
- 직접 조합 모드에서는 어떤 조합이든 허용 (사용자 자유)

### 8-2) 시각적 충돌 방지 (필수)
- **Front × Top:** 앞머리가 정수리를 덮으면 Top이 안 보임 → 메시 voxel 충돌 검사
- **Front × Side:** 앞머리 옆 부분과 옆머리 시작 부분 메시 겹침 → 알파 블렌딩 / 살짝 안쪽으로 정렬
- **Back × Side:** 긴 옆머리와 뒷머리가 어깨에서 겹침 → 자연스러움 (실제 머리도 그러함)

### 8-3) 데이터 기반 (Phase 2)
사용자가 직접 조합한 결과를 "내가 좋아함"으로 표시 → 학습 데이터로 호환성 점수 모델 fine-tune.

## 12. 핵심 요약
