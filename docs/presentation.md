# 애니메이션 스타일 가상 아바타 자동 생성 시스템

## 1. 프로젝트 주제 및 목표

레퍼런스 이미지 한 장으로부터 애니메이션 스타일의 3D 아바타를 자동 생성하는 웹 기반 시스템.

사용자가 원하는 캐릭터 이미지를 업로드하면, 얼굴 형태(슬라이더 키값), 텍스처(피부·눈·눈썹 색상), 헤어스타일을 자동으로 추출·적용하여 VRM 아바타를 생성한다.

**최종 목표**: 전문 지식 없이도 누구나 자신만의 애니메이션 아바타를 만들 수 있는 파이프라인 구축

---

## 2. 문제 정의

### 기존 아바타 생성 방식의 한계

| 문제 | 설명 |
|------|------|
| **수동 조작의 번거로움** | VRoid Studio 등 기존 도구는 29개 이상의 슬라이더를 사용자가 직접 조절해야 함 |
| **전문성 요구** | 텍스처 제작에 그래픽 편집 도구(Photoshop 등) 숙련도 필요 |
| **시간 소요** | 하나의 아바타를 만드는 데 수십 분~수 시간 소요 |
| **레퍼런스 재현 어려움** | 원하는 캐릭터와 유사한 아바타를 만들기 위한 시행착오 반복 |

### 우리의 접근

> **"이미지 한 장 업로드 → 아바타 자동 생성 → 미세 조정"**

레퍼런스 이미지에서 얼굴 특징을 자동 추출하고, 이를 VRM 모프 타겟 + 텍스처에 매핑하여 초기 아바타를 즉시 생성한 뒤, 사용자가 슬라이더로 미세 조정할 수 있게 한다.

## 3. 프로젝트 진행사항

### 마일스톤 달성도

| 단계 | 내용 | 상태 |
|------|------|------|
| **M1** | 기반 인프라 — face-feature 파이프라인 구축 (ADF 28pt → 29 Avatar Keys) | ✅ 완료 |
| **M2** | 프론트엔드 통합 — 타입 정의, 컴포넌트, 상태 관리, Mock 데이터 | ✅ 완료 |
| **M3** | 헤어/물리 시스템 — SpringBone 추출, 헤어 색상 매칭, 프리셋 | ✅ 완료 |
| **M4** | 텍스처 파이프라인 — Gemini 기반 특징 추출, UV 마스크 텍스처 생성 | ✅ 완료 |
| **M5** | 파이프라인 통합 — face-keys API + texture API, Mock→실제 전환 | ✅ 완료 |
| **M6** | 품질 개선 — Hue Shift, 아이섀도, Gemini 보정, Kanosawa fallback | ✅ 완료 |
| **M7** | ADF Docker 서버 — 로컬 Docker 환경 구성 및 동작 확인 | ✅ 완료 |

### 주요 성과

- **이미지 업로드 → 아바타 생성 전 과정 자동화** (face-keys + 텍스처 + 헤어 매칭 병렬 실행)
- **29개 Avatar Key** 자동 추출 및 VRM 모프 타겟 즉시 적용
- **7종 텍스처** 자동 생성 (얼굴, 눈썹, 아이라인, 홍채, 흰자, 하이라이트, 입)
- **3종 템플릿** 자동 분류 (cute / slim / mature, 코사인 유사도 기반)
- **ADF 서버 없는 환경** 대응 (Kanosawa 24pt fallback + damping)

### 3가지 핵심 파이프라인

**① 얼굴 슬라이더 추출 (Face Keys)**
- ADF(anime-face-detector)로 28개 얼굴 랜드마크 검출
- 랜드마크 좌표 → 기하학적 비율 계산 → 29개 Avatar Key 매핑
- OpenCV HoughCircles로 동공 크기 검출
- 코사인 유사도 기반 cute/slim/mature 템플릿 자동 선택

**② 텍스처 자동 생성 (Texture Pipeline)**
- Gemini API로 정성적 특징 추출 (피부색, 눈 색상, 아이섀도, 속눈썹 등)
- UV 마스크 기반 베이스 텍스처에 색상·디테일 적용
- Hue Shift 방식으로 원본 텍스처 색조 변화 보존
- 홍채 5방향 보간 + 방사형 링 구조로 자연스러운 눈동자 표현

**③ 헤어 자동 매칭 (Hair Matching)**
- 이미지 상단 40%에서 헤어 색상 추출 (배경 필터링 + 클러스터링)
- 프리셋 헤어 모델과 색상 유사도 비교
- SpringBone 물리 데이터 포함 GLB 자동 장착

## 7. 구현 내용

### 7-1. 이미지 업로드 및 자동 처리

- 드래그 앤 드롭 또는 파일 선택으로 레퍼런스 이미지 업로드
- 업로드 즉시 **face-keys 추출**, **텍스처 생성**, **헤어 매칭** 3개 파이프라인 병렬 실행
- 처리 완료 시 VRM 모델에 자동 적용

### 7-2. 29개 얼굴 슬라이더 자동 추출

ADF 28포인트 랜드마크에서 기하학적 비율을 계산하여 29개 Avatar Key로 변환:

| 카테고리 | 키 수 | 예시 |
|----------|--------|------|
| 눈 (Eye) | 14개 | 크기, 간격, 기울기, 눈꺼풀 곡률, 동공 크기 |
| 눈썹 (Brow) | 5개 | 간격, 높이, 기울기, 폭, 두께 |
| 코 (Nose) | 3개 | 높이, 폭, 인중 길이 |
| 입 (Mouth) | 3개 | 폭, 높이, 꼬리 올림 |
| 얼굴형 (Face) | 4개 | 턱선, 볼, 둥근 정도, 턱 폭 |

### 7-3. 7종 텍스처 자동 생성

Gemini API가 추출한 색상·속성 정보를 UV 마스크 기반 베이스 텍스처에 적용:

| 텍스처 슬롯 | 적용 내용 |
|-------------|----------|
| Face (피부) | 피부색 + 쉐이딩 강도 |
| Eyebrow (눈썹) | 눈썹색 + 그라데이션 |
| Eyeline (아이라인) | 아이라인색 + 아이섀도 + 속눈썹 강도 |
| Pupil (홍채) | 5방향 색상 보간 + 방사형 링 + 동공 크기 |
| Eye White (흰자) | 흰자 색상 |
| Eye Highlight (하이라이트) | 하이라이트 색상 |
| Mouth Inside (입 안) | 입 안쪽 색상 |

### 7-4. 헤어 자동 매칭 및 물리

- 이미지 상단 40%에서 헤어 색상 클러스터링
- 5종 프리셋 헤어와 색상 유사도 비교 → 자동 추천
- SpringBone 물리 데이터 포함 GLB로 자연스러운 머리카락 움직임

### 7-5. 에디터 기능

- 29개 슬라이더 수동 미세 조정
- 텍스처 색상/질감 편집
- 헤어/의상 교체
- 버전 저장/복원 (최대 5개, Undo/Redo 지원)
- cute/slim/mature 템플릿 전환

## 8. 핵심 이슈 및 문제점

### 해결된 이슈

| 이슈 | 원인 | 해결 |
|------|------|------|
| 텍스처가 밋밋함 | `adjust_hue()`가 원본 Hue를 통째로 교체 | Hue Shift 방식(`adjust_hue_v2`)으로 원본 색조 변화 보존 |
| Gemini 특징 미적용 | 아이섀도, 속눈썹, 그라데이션 데이터가 코드에서 무시됨 | 조건부 게이팅으로 활성화 |
| Face_Cheek 항상 0 | 2D 입력 시 side depth 없어 0.0으로 덮어쓰기 | 2D 윤곽 계산값 유지하도록 수정 |
| 턱/얼굴형 부정확 | 순수 기하학적 비율만 사용 | Gemini face_shape/chin_shape/jaw_prominence 30% 블렌딩 |
| macOS에서 ADF 사용 불가 | ADF가 Linux(WSL) 전용 | Docker x86 에뮬레이션 + Kanosawa fallback |
| mmcv/numpy 호환성 | mmpose가 mmcv≤1.7.0 요구, numpy 2.x 비호환 | 버전 핀 (mmcv 1.7.0, numpy<2) |

### 현재 한계점

| 한계 | 설명 |
|------|------|
| **동공 검출 불안정** | OpenCV HoughCircles의 반지름 추정이 불안정 (특히 애니메이션 눈) |
| **눈꺼풀 처짐 미측정** | Eye_TopLidDown, Eye_LowerLidUp 항상 0 (애니 눈은 보통 완전 개방) |
| **캘리브레이션 데이터 부족** | 정규화 범위(lo/hi)가 26개 이미지 기반 → 충분하지 않음 |
| **x86 에뮬레이션 속도** | Apple Silicon에서 ADF Docker가 느림 (첫 추론 ~30초) |
| **Kanosawa damping** | Fallback 시 모든 슬라이더 값이 45% 압축되어 특징이 덜 뚜렷함 |

## 10. 피드백 요청사항

1. **슬라이더 키값 정확도**: 레퍼런스 이미지와 생성된 아바타 간 유사도가 체감상 어느 정도인지
2. **텍스처 품질**: 색상 재현도 및 디테일(아이섀도, 그라데이션 등)의 자연스러움
