## Slide 2. 프로젝트 소개 — 우리는 무엇을 만드는가

**한 줄 정의:** AI가 사진을 3D로 변환하고, 표준 VTuber 템플릿 슬라이더로 편집해, VRM 파일로 즉시 활동할 수 있는 웹 SaaS 플랫폼.

**핵심 가치 4가지**

| 가치 | 내용 |
|---|---|
| 접근성 혁신 | 사진 1장 → 30초 안에 3D VTuber 아바타, 전문 지식 불필요 |
| 편집 자유도 | 60+ 블렌드셰이프 슬라이더로 표정·체형·의상 미세 조정 |
| 저작권 보장 | 생성 아바타의 완전한 저작권을 사용자가 소유 (VRM 익스포트) |
| 플랫폼 호환 | Unity + UniVRM 기반 → OBS, VSeeFace 등 즉시 연동 |

**비전:** "수백만 원·수개월이 걸리던 VTuber 아바타 제작을, AI와 표준 템플릿으로 15분·2.5만 원에 가능하게 한다."

**발표 멘트:**
한 줄로 말씀드리면 "사진 한 장 → 30초 3D 생성 → 슬라이더 편집 → VRM 다운로드"가 끊김 없이 이어지는 웹 서비스입니다.

## Slide 3. 배경 & MVP 범위 — 왜·무엇을 만드는가

**왜 (Pain Point)**
- 한국 VTuber 활동 인구 약 5만 명, 글로벌 가상 인플루언서 시장 $18B → $270B (CAGR ~57%)
- 오리지널 아바타 의뢰 비용 80~200만 원, 대기 2~3개월 → **진입 장벽이 시장 성장을 못 따라감**
- 기성 AI 3D 도구(Meshy/Tripo 등)는 **생성만 가능**, VTuber용 편집·VRM 출력 부재

**무엇을 만드는가 — MVP 정의**
> "사진 → 3D 생성 → 슬라이더 편집 → 버전 관리 → 유사도 검증 → VRM 익스포트"가 끊김 없이 동작하는 것

**Must-Have (MVP 포함)**
이미지 업로드 / VARCO 3D 생성 / 슬라이더 편집(20~30종) / VRM 익스포트 + 자동 리깅 / 버전 5개 관리 / 유사도 검증 게이트(코사인 85%) / OAuth 인증 / 결제(건당) / NSFW·실존인물 모방 필터

**Phase 2 이후 (의도적 제외)**
자연어 편집 / 텍스처·의상 편집 / FBX·GLB 익스포트 / 라이브 스트리밍 즉시 연동 / 마켓플레이스 / 모바일

**발표 멘트:**
MVP 범위는 의도적으로 좁혔습니다. 자연어 편집·의상 생성처럼 매력적이지만 리스크가 큰 기능은 Phase 2로 미루고, "사용자가 처음 VTuber로 데뷔한다"는 핵심 가치만 끝까지 완성하는 것이 이번 학기 목표입니다.

## Slide 4. 시스템 구조 — 9단계 파이프라인 & 기술 스택

**9단계 파이프라인**

```
[1] 이미지 업로드 (NSFW 필터)  → [2] VARCO GLB 생성 → [3] 정면/측면 렌더
                                                                ↓
[6] Material 커스터마이즈 ← [5] Bone Scale 조정 ← [4] Morph 파라미터 매핑
        ↓
[7] 유사도 검증 게이트 → [8] Three.js 실시간 3D 편집 → [9] Unity VRM 익스포트
```

**핵심 설계 결정:** VARCO 결과는 "참조용", 최종 편집은 검증된 표준 VTuber 템플릿(cute / slim / mature)에서 수행 → VRM 호환성 100% + 메시 깨짐 방지.

**기술 스택 요약**

| 영역 | 기술 |
|---|---|
| 프론트엔드 | Next.js + TypeScript + Tailwind + shadcn/ui |
| 3D 에디터 | Three.js + react-three-fiber + @pixiv/three-vrm |
| AI 3D 생성 | VARCO 3D API (NCSoft) |
| 특징 추출 | Python + MediaPipe + OpenCV |
| 백엔드 | NestJS + BullMQ + Redis |
| DB / 스토리지 | PostgreSQL + Prisma / S3-compatible |
| VRM 출력 | Unity + UniVRM (headless) |

**발표 멘트:**
이 파이프라인의 핵심은 "AI 생성 결과를 그대로 쓰지 않고 검증된 템플릿에 매핑한다"는 점입니다. 이 9단계 중 현재 어디까지 동작하는지를 다음 슬라이드에서 보여드리겠습니다.

**시장·운영 이슈**
- "AI 품질이 작가 수준에 못 미친다" 인식 → "빠르고 저렴한 시작점" 포지셔닝으로 대응
- 샘플→결제 전환율 8% 미달 시 → 워터마크 강도·가격(24,900원) A/B 테스트
- 5명 인력으로 Phase 1 일정(총 12주) 빠듯 → Sprint 0 결과로 MVP 범위 한 차례 재조정 완료

**법·저작권 이슈**
- AI 저작권 = 사용자 입력(이미지) + 창작적 선택(슬라이더 조작) → 사용자 귀속으로 정리
- 실존 인물 모방 차단: 코사인 유사도 85% 게이트 + DMCA Notice & Takedown 프로세스

**논의 요청 사항(피드백 받고 싶은 부분)**
1. 템플릿 3종으로 시작하는 범위가 적절한지, 추가해야 한다면 어떤 스타일이 우선인지
2. 결제 전환을 위한 무료 샘플 정책(워터마크 vs 시간 제한) 중 더 효과적인 안
3. 베타 50명 모집 시 어떤 채널이 가장 실효적인지 (SOOP / 디스코드 / 트위터)

**발표 멘트:**
이 슬라이드가 오늘 발표의 가장 중요한 부분입니다. Q&A 때 여기 적힌 세 가지 논의 요청 사항에 대해 피드백 주시면 정말 큰 도움이 됩니다.

## Slide 9. 마무리 & Q&A

**핵심 메시지 3가지**
1. **시장과 문제는 명확하다** — VTuber 시장은 폭발적으로 성장하지만 진입장벽(시간·비용·저작권)은 그대로다.
2. **포지션은 비어 있다** — AI 생성 + 템플릿 슬라이더 편집 + VRM 출력 + 사용자 저작권을 모두 충족하는 제품은 현재 없다.
3. **MVP의 가장 큰 기술 리스크는 PoC로 해소되었다** — 현재 프로토타입 v0.5에서 핵심 흐름이 end-to-end 동작하며, 남은 6주는 결제·인증·QA·베타에 집중한다.

**Q&A 예상 질문 카드 (내부용)**
1. 템플릿 3종으로 충분한가? → MVP 범위 정의 + Phase 2 확장 계획
2. VARCO 의존도 낮추는 전략은? → Meshy/Tripo 멀티 벤더 백업, 자체 모델 장기 투자
3. 저작권 이슈는 어떻게 정리되는가? → 사용자 입력+창작적 선택 → 사용자 귀속 + DMCA 프로세스
4. 결제 전환율 8% 근거는? → SaaS 크리에이터 툴 평균 + 무료 샘플 워터마크 전략
