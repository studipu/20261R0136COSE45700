# Product Requirements Document - virtual_avatar

**Author:** Seungu
**Date:** 2026-04-10
**Version:** v2.0 (시장분석 + BM + 기술 파이프라인 통합)

## Executive Summary

사진 한 장으로 누구나 자신만의 오리지널 3D 아바타를 만들고, 표준 VTuber 템플릿 기반 슬라이더(morph target/bone scale/material)로 편집하며, 완전한 저작권을 소유한 채 즉시 VTuber 활동을 시작할 수 있는 웹 기반 SaaS 플랫폼.

현재 오리지널 3D 아바타를 확보하려면 전문 작가에게 의뢰해야 한다. 비용은 수십만~수백만 원, 대기 기간은 수주~수개월, 수정 요청 시 작업 큐에 밀려 피드백 반영까지 추가 수일이 소요된다. 비오리지널(기성) 아바타는 저렴하지만 사용권만 부여되어 수익화 제한, 수정 불가, 비독점 사용, DMCA 리스크라는 구조적 한계를 갖는다.

본 서비스는 VARCO 3D(Image-to-3D)로 참조 모델을 생성하고, 내부 표준 VTuber 템플릿(cute/slim/mature)에 **morph target/bone scale/material** 기반 편집 엔진을 결합하여, 이 세 가지 병목(비용·대기·수정 속도)을 동시에 제거한다. 사용자가 사진을 업로드하면 VARCO가 참조용 GLB를 생성하고, MediaPipe로 추출한 특징값이 슬라이더 초기값으로 매핑되어 직관적 조작으로 1초 이내에 편집 결과를 확인할 수 있다. AI 생성 아바타이므로 저작권은 사용자에게 귀속되며, 작가-의뢰인 간 저작권 분쟁 구조 자체가 소멸한다.

> **핵심 설계 결정:** VARCO 출력은 참조용(특징 추출 소스)이며 최종 제품이 아님. 실제 편집은 내부 표준 VTuber 템플릿에서 수행됨. 이는 품질 일관성과 VRM 호환성을 보장하기 위한 의도적 아키텍처 선택.

### What Makes This Special

**표준 VTuber 템플릿 + AI 참조 기반 슬라이더 편집은 현재 어떤 경쟁 플랫폼도 제공하지 않는다.** ZEPETO, VRoid Studio, Ready Player Me 등 기존 아바타 플랫폼은 프리셋 조합 방식에 머물러 있고, Meshy·Tripo·Rodin 등 AI 3D 생성 플랫폼은 생성만 지원하며 편집 기능이 없다. MetaHuman은 고품질이나 게임 엔진 종속적이며 VTuber 용도에 부적합하다.

핵심 차별점:
- **9단계 통합 파이프라인**: 업로드→VARCO 참조 생성→특징 추출→템플릿 매칭→슬라이더 편집→VRM 익스포트까지 단일 플로우
- **완전한 저작권 소유**: AI 생성 모델이므로 사용자가 저작권자. 수익화·2차 창작·독점 사용 모두 가능
- **1초 이내 편집 반응**: morph target/bone scale 기반으로 리깅/블렌드셰이프 보존하며 실시간 피드백
- **VTuber 특화 출력**: VRM 익스포트(MVP, Unity+UniVRM), FBX/GLB 확장(Phase 2), 자동 리깅, 블렌드셰이프 매핑 내장

기술 로드맵: MVP는 표준 템플릿 + morph target/bone scale 슬라이더 기반으로 빠르고 안정적인 편집을 제공하고, Phase 2에서 HeadEvolver/DaCapo 기반 자유형 텍스트 편집으로 확장, Phase 3에서 MeshLLM 기반 네이티브 3D 이해·편집으로 진화한다.

## User Journeys

### Journey 1: 하늘 — "첫 오리지널 아바타로 데뷔하기"
*Primary User, Success Path*

**페르소나:** 22세 대학생. VTuber 데뷔를 꿈꾸지만 오리지널 3D 아바타 의뢰 비용(80~200만원)을 감당할 수 없다. BOOTH에서 기성 아바타(3~5만원)를 사봤지만 사용권만 있어 수익화 제한이 걸리고, 같은 아바타를 쓰는 다른 VTuber와 마주칠까 불안하다.

**Opening Scene:**
하늘은 트위치에서 VTuber 방송을 보며 "나도 하고 싶다"고 생각한다. 작가에게 견적을 문의하니 최소 80만원, 대기 3개월. 알바비로는 엄두가 나지 않는다. BOOTH 기성 아바타는 저렴하지만 "수익화 시 별도 라이선스 필요", "2차 수정 금지" 조건이 붙어있다.

**Rising Action:**
SNS에서 "사진 한 장으로 3D 아바타 만들기" 광고를 본다. 반신반의하며 서비스에 접속. 좋아하는 애니 캐릭터 스타일의 참고 이미지를 업로드한다. 3분 후, 화면에 3D 아바타가 나타난다. "오... 진짜 나왔어?" 기대 반 불안 반으로 모델을 돌려본다.

코가 좀 크다. 눈매가 원하는 느낌이 아니다. 슬라이더 패널에서 `코 높이`를 줄이고 `눈 크기`를 키운다. 변경이 즉시 반영된다. 작가에게 "코 좀 작게 해주세요" 하고 3일을 기다리던 과거와 완전히 다르다. 20분간 이것저것 만져보며 3개 버전을 저장한다.

**Climax:**
버전 비교 화면에서 세 버전을 나란히 놓고 "이거다!" 하는 순간. VRM 익스포트 버튼을 누르자 자동 리깅, 블렌드셰이프 매핑이 완료된 파일이 다운로드된다. VSeeFace에서 열어보니 웹캠에 맞춰 아바타가 눈을 깜빡이고 입을 벌린다.

**Resolution:**
하늘은 그 주말 첫 방송을 시작한다. 완전한 저작권을 가진 오리지널 아바타. 수익화 제한 없음. 수정하고 싶으면 언제든 서비스에 돌아와 슬라이더를 조정하면 된다. 의뢰 비용 0원, 소요 시간 30분.

> **도출 요구사항:** 이미지 업로드 UI, 3D 생성 프로그레스 표시, 슬라이더 편집 패널(20-30개 파라미터), 버전 저장/비교, VRM 익스포트, 블렌드셰이프 자동 매핑, 저작권 안내 페이지

### Journey 2: 미카 — "기존 아바타를 대체할 새 아이덴티티 만들기"
*Primary User, Edge Case — 리브랜딩 + 유사도 게이트 충돌*

**페르소나:** 26세 한일 양국 활동 VTuber. 현재 아바타의 계약이 "사용권만 양도"여서, 굿즈 판매·2차 창작 수익화에 제약이 있다. 새 오리지널 아바타로 리브랜딩하려 하지만, 기존 팬이 알아볼 수 있는 유사한 느낌을 유지하고 싶다.

**Opening Scene:**
미카는 1년간 쓴 아바타가 있지만 저작권이 작가에게 있어 굿즈를 만들 수 없다. "이 캐릭터를 저작권까지 사고 싶다"고 작가에게 문의하니 추가 300만원. 차라리 새로 만들기로 결심한다.

**Rising Action:**
서비스에 접속하여 자신의 현재 아바타 이미지를 참고로 업로드한다. 3D 아바타가 생성되지만 — **유사도 검증 게이트가 경고를 표시한다.** "기존 등록된 캐릭터와 유사도 87%. 저작권 분쟁 위험이 있습니다." 미카는 당황하지만, 시스템이 "어떤 부분을 변경하면 유사도를 낮출 수 있는지" 가이드를 제공한다.

헤어스타일 슬라이더를 변경하고, 눈 색상을 바꾸고, 얼굴형을 조정한다. 유사도가 62%로 내려간다. 게이트 통과. "느낌은 비슷하지만 법적으로 완전히 다른 캐릭터"가 완성된다.

**Climax:**
VRM 익스포트 후 방송에서 새 아바타를 공개. 팬들은 "느낌이 비슷해서 좋다"고 반응. 미카는 이제 저작권자이므로 아크릴 스탠드, 스티커 등 굿즈 판매를 시작한다.

**Resolution:**
리브랜딩 성공. 기존 팬 유지 + 수익화 채널 확장. 의뢰 비용 0원, 법적 리스크 제거.

> **도출 요구사항:** 유사도 검증 게이트 (코사인 유사도 + 시각적 경고), 유사도 감소 가이드, 편집 전/후 유사도 실시간 표시, 저작권 확인서 발급

### Journey 3: 진수 — "소속 VTuber 10명 아바타 일괄 제작"
*B2B User, Agency Workflow*

**페르소나:** 34세 VTuber 에이전시 대표. 신규 VTuber 10명 동시 데뷔를 준비 중. 기존 방식(작가 10명 개별 의뢰)으로는 총 비용 1,500만원+, 소요 기간 4개월+. 통일된 퀄리티 보장도 어렵다.

**Opening Scene:**
진수는 신규 VTuber 계약을 체결했지만 아바타 제작이 병목이다. 작가 A는 2개월 대기, 작가 B는 스타일이 다르다. 10명의 아바타를 비슷한 품질로, 빠르게, 합리적 비용으로 만들어야 한다.

**Rising Action:**
서비스의 비즈니스 플랜에 가입한다. 각 VTuber에게 참고 이미지를 제출받아 10개 프로젝트를 생성한다. 아바타가 생성되면, 진수가 1차로 슬라이더를 조정하고 각 VTuber에게 편집 링크를 공유한다. VTuber들이 직접 자기 아바타를 미세 조정한다.

진수는 대시보드에서 10명의 진행 상황을 확인한다. 7명 완료, 2명 편집 중, 1명 미시작. 완료된 아바타를 일괄 VRM 익스포트한다.

**Climax:**
10일 만에 10명의 오리지널 아바타가 모두 완성된다. 기존 4개월 → 10일. 비용은 1/10 이하. 모든 아바타의 저작권이 에이전시에 귀속된다.

**Resolution:**
10명 동시 데뷔 성공. 진수는 다음 기수 모집 시에도 동일 프로세스를 반복할 수 있다는 확신을 갖는다.

> **도출 요구사항:** 비즈니스 플랜 (다중 프로젝트), 편집 링크 공유, 프로젝트 대시보드 (진행 상황 모니터링), 일괄 익스포트, 팀/권한 관리

### Journey 4: 운영자 — "서비스 품질과 법적 안전성 관리"
*Admin/Operations User*

**페르소나:** 서비스 운영팀. 유사도 게이트 관리, 신고 처리, 사용량 모니터링, 서비스 품질 유지를 담당한다.

**Opening Scene:**
서비스 런칭 후 일일 아바타 생성 수가 200건을 넘기 시작한다. 유사도 게이트에서 걸린 건 중 "오탐(false positive)" 신고가 들어오고, VARCO API 응답 시간이 간헐적으로 느려진다.

**Rising Action:**
운영자는 관리 대시보드에 접속한다.
- **유사도 게이트 관리:** 임계값(현재 85%)을 확인하고, 오탐 신고 건을 리뷰한다. 해당 아바타 쌍을 직접 비교하고 "오탐 확인 → 게이트 통과 승인"을 처리한다.
- **사용량 모니터링:** 일일 생성 수, API 호출량, 평균 생성 시간, 에러율을 대시보드에서 확인한다. VARCO API 응답 시간이 P95에서 8분으로 올라간 것을 감지, 알림을 설정한다.
- **신고 처리:** 사용자가 "다른 VTuber 아바타를 복제한 것 같다"고 신고한 건을 조사한다. 유사도 로그를 확인하고, 필요시 해당 아바타를 비공개 처리한다.

**Resolution:**
운영자는 매일 30분 내로 핵심 지표를 확인하고, 이슈를 처리할 수 있다. 임계값 조정은 월 1회 데이터 기반으로 리뷰한다.

> **도출 요구사항:** 운영 대시보드 (사용량/성능 지표), 유사도 게이트 임계값 설정, 오탐 리뷰/승인, 신고 접수/처리 워크플로, 아바타 비공개 처리, API 성능 알림

## Domain-Specific Requirements

### 저작권 & 법적 요구사항

- **AI 생성물 저작권:** 한국 저작권법상 AI 생성물의 저작권 귀속은 아직 명확한 판례가 없음. 이용약관에서 "사용자가 입력(참고 이미지)을 제공하고 창작적 선택(슬라이더 조작)을 하므로 사용자에게 권리 귀속"으로 설계. 약관 법률 검토 필수
- **유사도 게이트 법적 근거:** 타인의 캐릭터와 유사한 아바타 생성 시 서비스 제공자의 책임 범위 명확화. 유사도 임계값(85%)의 법적 방어 가능성 검토 필요
- **참고 이미지 권리:** 사용자가 업로드하는 참고 이미지의 저작권 확인 절차. 타인의 저작물을 무단 사용한 경우 서비스의 면책 조항 필요
- **DMCA/저작권 침해 신고 대응:** 제3자의 저작권 침해 신고 접수 및 처리 프로세스 (Notice & Takedown)

### 기술 제약사항

- **VARCO API 의존성:** 핵심 3D 생성 기능이 외부 API(NCSoft VARCO)에 의존. API 중단/변경 시 서비스 전체에 영향. SLA 확보 및 fallback 전략 필요
- **3D 포맷 호환성:** VRM 0.x → 1.0 전환기. MVP에서 양쪽 버전 모두 지원 필요. Phase 2 FBX/GLB 확장 시 리깅/블렌드셰이프 데이터 무결성 보장
- **템플릿 다양성:** MVP에서 3종(cute/slim/mature)만 제공. 사용자 니즈 대비 부족할 수 있으며, Phase 2에서 5종+ 확장 필요
- **VARCO→템플릿 특징 매핑:** MediaPipe로 추출한 특징값을 템플릿 슬라이더에 매핑하는 정확도가 사용자 만족도에 직결. 매핑 테이블 수동 튜닝 필요
- **Unity headless 서버:** VRM 변환을 위한 Unity headless 서버 운영 복잡도. Docker 컨테이너화, 오토스케일링 전략 필요
- **헤어/의상 제한:** AI 기반 헤어/의상 생성 불가 → VRoid 스타일 프리셋으로 대응. Phase 2에서 텍스처 생성 AI 도입 검토
- **GPU 리소스:** VARCO 생성(Stage 2), 렌더링(Stage 3), VRM 변환(Stage 9)은 서버사이드 비동기 처리. 슬라이더 편집(Stage 7)은 클라이언트사이드(WebGL/Three.js). 클라이언트 GPU 최소 사양 정의 필요

### 데이터 프라이버시

- **참고 이미지 처리:** 사용자 업로드 이미지는 3D 생성 후 보관 정책 명확화 (즉시 삭제 vs 재생성용 보관)
- **3D 모델 저장:** 생성된 아바타 데이터의 저장 위치, 암호화, 보관 기간
- **개인정보보호법 준수:** 한국 개인정보보호법 기준. 얼굴 사진 업로드 시 생체정보 해당 여부 검토 (실사 사진이 아닌 일러스트 참고 이미지 중심이므로 리스크 낮음)

- **신고 시스템:** 사용자 간 저작권 침해/부적절 콘텐츠 신고 및 처리

## Innovation & Novel Patterns

### Detected Innovation Areas

**1. AI 참조 + 표준 템플릿 편집의 대중화 (핵심 혁신)**
VARCO AI 생성을 참조 소스로 활용하고, 검증된 VTuber 템플릿의 morph target/bone scale을 슬라이더로 제어하는 최초의 비전문가용 제품. 기존 경쟁사는 프리셋 조합(VRoid, ZEPETO) 또는 생성만(Meshy, Tripo)에 머물러 있다. 슬라이더 기반 연속적 편집은 프리셋의 이산적(discrete) 선택과 근본적으로 다른 UX를 제공한다.

**2. 9단계 통합 파이프라인**
현재 3D 아바타 제작은 생성(VARCO/Meshy), 편집(Blender), 리깅(Unity), 익스포트(VRM 변환)가 별도 도구·별도 전문성을 요구하는 분절된 워크플로. 업로드→VARCO 생성→렌더→특징추출→템플릿매칭→초기화→편집→저장→VRM 익스포트를 단일 웹 인터페이스로 통합하는 것은 VTuber 도구 시장에서 최초.

**3. AI 기반 저작권 패러다임 전환**
전통적 아바타 커미션 모델(작가 저작권 → 사용권만 양도)을 AI 생성으로 대체하면, 저작권이 사용자에게 자연스럽게 귀속된다. 이는 단순한 기능 혁신이 아니라 크리에이터 경제의 구조적 변화.

### Market Context & Competitive Landscape

**시장 규모:**
- 일본 VTuber 시장: **800億円** (~8,000억 원, 2024)
- 글로벌 가상 인플루언서 시장: **$18.19B → $270.61B** (2024→2030, CAGR ~57%)
- 한국 VTuber 추정 인구: ~50,000명 (활동 중 기준)
- 한국 스트리밍 생태계: CHZZK 점유율 **39%**, SOOP 연간 시청 **110억 시간**

**경쟁사 4분류:**
- **프리셋 기반 에디터** (VRoid, ZEPETO): 무료/접근성 강점, 오리지널리티 한계
- **AI 3D 생성** (Meshy, Tripo, Rodin): 생성만 지원, 편집 불가, VTuber 미특화
- **전문 3D 툴** (Blender, Maya): 자유도 최고, 전문 지식 필수
- **커미션 마켓** (BOOTH, Skeb, 크몽): 작가 퀄리티, 고비용·장기 대기

- **학계:** HeadEvolver가 텍스트 가이드 mesh 변형을 시연했으나 제품화되지 않음

**Virtual Avatar가 유일하게 AI 참조 생성 + 템플릿 기반 슬라이더 편집 + VTuber 특화 출력(VRM)을 모두 충족하는 포지션.**

### Validation Approach

| AI 저작권 법률 불확실성 | 이용약관에 "현행법 변경 시 조항 수정" 유연 조항 포함. 법률 자문 분기별 리뷰 |

## SaaS + Web App Specific Requirements

### Project-Type Overview

3D 아바타 생성·편집·익스포트를 제공하는 웹 기반 SaaS. WebGL 기반 3D 뷰어/편집기가 핵심이므로 SPA(Single Page App) 아키텍처를 채택하며, 데스크톱 브라우저 전용으로 MVP를 출시한다.

### 테넌트 & 권한 모델

**MVP (단일 테넌트):**
- 개인 계정 기반. 사용자별 프로젝트(아바타) 목록 관리
- 역할: 일반 사용자 / 운영자(어드민)
- 프로젝트 단위 접근 제어 (본인 프로젝트만 접근)

**Growth (멀티테넌트 확장):**
- 조직/팀 계정 추가 (에이전시 워크플로)
- 역할 확장: 조직 관리자 / 편집자 / 뷰어
- 편집 링크 공유 (조직 내 협업)
- 프로젝트 대시보드 (진행 상황 모니터링)

### 과금 모델 (MVP 단순화)

#### 요금 체계

| 단계 | 상품명 | 가격 | 포함 내용 |
|------|--------|------|----------|
| **무료** | Free Sample | 0원 | 이미지 업로드 → 3D 생성 → 워터마크 미리보기 (편집 불가) |

8. Param Save → JSON 스냅샷 저장
9. Unity VRM Export → UniVRM 변환 + 자동 리깅 (Unity headless 서버)

## Project Scoping & Phased Development

### MVP Strategy & Philosophy

**MVP 접근: Problem-Solving MVP (문제 해결형)**
"오리지널 3D 아바타를 빠르고 저렴하게 만들 수 없다"는 핵심 문제를 최소 기능으로 검증한다. 완벽한 제품이 아니라, 사용자가 "이걸로 충분히 VTuber를 시작할 수 있다"고 말하는 최소 수준을 목표로 한다.

**Resource Requirements:**

| 역할 | 인원 | 핵심 업무 |
|------|------|----------|
| 3D/템플릿 엔지니어 A | 1명 | 템플릿 제작(cute/slim/mature), morph target/bone scale 정의, Unity VRM 변환 파이프라인 |
| 3D/뷰어 엔지니어 B | 1명 | Three.js + react-three-fiber 뷰어, @pixiv/three-vrm, 렌더링 최적화(LOD) |
| 프론트엔드 | 1명 | Next.js + TypeScript, UI/UX, 슬라이더 패널 (Zustand), 버전 관리 UI |
| 백엔드 | 1명 | NestJS API, BullMQ/Redis 파이프라인, Python MediaPipe 연동, 크레딧/결제 |
| PM/디자인 | 1명 | 제품 관리, UX 설계, QA, VTuber 커뮤니티 리서치 |

**총 5명, 3D 파트에 2명 집중 배치** — 템플릿 품질 + 뷰어 성능이 최고 리스크이므로 3D A(템플릿/Unity)와 B(뷰어/렌더링)로 책임 분리.

## Functional Requirements

### 아바타 생성 — 파이프라인 Stage 1-6 (Avatar Generation)

- FR1: 사용자는 참고 이미지를 업로드하여 3D 아바타 생성을 요청할 수 있다 (Stage 1)
- FR2: 시스템은 업로드된 이미지를 기반으로 VARCO 참조용 GLB를 생성한다 (Stage 2)
- FR3: 사용자는 생성 진행 상태를 실시간으로 확인할 수 있다 (BullMQ + WebSocket)
- FR4: 시스템은 참조 GLB에서 정면/측면 렌더링을 수행한다 (Stage 3)
- FR5: 시스템은 MediaPipe + OpenCV로 얼굴/체형 특징을 추출한다 (Stage 4)
- FR6: 시스템은 추출된 특징 기반으로 최적 템플릿(cute/slim/mature)을 자동 선택한다 (Stage 5)

- FR8: 시스템은 추출 특징값을 슬라이더 초기값으로 매핑한다 (Stage 6)

### 아바타 편집 — 파이프라인 Stage 7-8 (Avatar Editing)

- FR9: 사용자는 morph target 슬라이더를 통해 얼굴 형태를 조정할 수 있다 (눈 크기, 코 높이, 턱 라인 등)
- FR10: 사용자는 bone scale 슬라이더를 통해 체형을 조정할 수 있다
- FR11: 사용자는 material 슬라이더를 통해 피부색, 눈동자 색, 머리 색 등을 변경할 수 있다
- FR12: 사용자는 VRoid 스타일 프리셋에서 헤어/의상을 선택할 수 있다
- FR13: 시스템은 슬라이더 조작 결과를 1초 이내에 3D 뷰어에 반영한다 (Stage 7)
- FR14: 사용자는 편집 중인 아바타의 버전을 저장할 수 있다 (최대 5개, Stage 8)
- FR15: 사용자는 저장된 버전들을 나란히 비교할 수 있다
- FR16: 사용자는 이전 버전으로 되돌릴 수 있다

### 익스포트 — 파이프라인 Stage 9 (Export)

- FR17: 사용자는 완성된 아바타를 VRM 포맷으로 익스포트할 수 있다 (Unity headless + UniVRM)
- FR18: 시스템은 익스포트 시 자동으로 humanoid 리깅을 적용한다
- FR19: 시스템은 익스포트 시 표정 블렌드셰이프(blink, joy, angry, sad, neutral + viseme)를 자동 매핑한다
- FR20: 사용자는 익스포트된 VRM 파일을 다운로드할 수 있다

- FR32: 시스템은 Edit 세션 시 29크레딧(2,900원)을 차감한다
- FR33: 시스템은 VRM Export 시 9크레딧(900원)을 차감한다
- FR34: 사용자는 결제 및 크레딧 사용 내역을 조회할 수 있다

### 콘텐츠 안전 (Content Safety)

- FR35: 시스템은 업로드된 참고 이미지에 대해 NSFW/부적절 콘텐츠 필터링을 수행한다 (Stage 1)
- FR36: 사용자는 다른 사용자의 아바타에 대해 저작권 침해 신고를 제출할 수 있다
- FR37: 시스템은 실존 인물 모방 아바타 생성을 감지한다 (Stage 4 특징 추출 활용)

### 운영 관리 (Administration)

- FR38: 운영자는 서비스 사용량 지표(일일 생성 수, 파이프라인 처리량, API 호출량, 에러율)를 대시보드에서 확인할 수 있다
- FR39: 운영자는 유사도 게이트 임계값을 조정할 수 있다
- FR40: 운영자는 오탐(false positive) 신고를 리뷰하고 게이트 통과를 승인할 수 있다
- FR41: 운영자는 신고된 아바타를 조사하고 비공개 처리할 수 있다
- FR42: 운영자는 API/파이프라인 성능 이상 시 알림을 받을 수 있다

### Security

- 모든 데이터 전송은 TLS 1.3으로 암호화한다
- 사용자 비밀번호는 bcrypt/argon2로 해싱하여 저장한다
- 결제 처리는 PG사 위임 (PCI-DSS 직접 인증 불필요). Stripe/토스페이먼츠 등 인증된 PG 사용
- OAuth 토큰은 서버사이드에서만 처리하며 클라이언트에 노출하지 않는다
- 사용자 업로드 이미지는 접근 제어된 스토리지에 저장한다 (본인만 접근)
- 생성된 3D 모델은 사용자별 격리된 스토리지에 저장한다
- API 엔드포인트에 rate limiting을 적용한다 (생성: 분당 5회, 편집: 분당 60회)
- 어드민 대시보드는 별도 인증 + IP 화이트리스트를 적용한다
- XSS, CSRF, SQL Injection 등 OWASP Top 10 방어를 적용한다

### Scalability

| 단계 | 동시 사용자 | 일일 생성 | 인프라 전략 |
|------|-----------|----------|-----------|
| MVP (3개월) | 50 | 200 | 단일 서버 + VARCO API |
| Growth (12개월) | 500 | 2,000 | 수평 확장, CDN, 큐 분리 |
| Expansion | 5,000+ | 20,000+ | 마이크로서비스, 멀티리전 |

- 3D 모델 스토리지는 오브젝트 스토리지(S3 호환)를 사용하여 용량 제한 없이 확장한다
- VARCO API 호출은 큐 시스템으로 관리하여 동시 호출 수를 제어한다
- 정적 에셋(Three.js, 텍스처 등)은 CDN으로 배포한다

### Integration

| 외부 시스템 | 용도 | SLA 요구 | Fallback |
|------------|------|----------|----------|
| VARCO 3D API | 참조용 GLB 생성 (Stage 2) | 99%+ 가용성, < 5분 | 에러 시 BullMQ 재시도 3회 + 사용자 알림. 장기 장애 시 Meshy/Tripo 백업 |
| MediaPipe (로컬) | 특징 추출 (Stage 4) | N/A (로컬 Python) | Python Worker 자체 실행 |
| Unity Headless | VRM 변환 (Stage 9) | 99%+, < 60초 | Docker 이중화, BullMQ 재시도 |
| PG사 (Stripe/토스) | 결제/크레딧 처리 | 99.9%+ | PG사 이중화 |
| OAuth Provider | 소셜 로그인 | 99.9%+ | 이메일/비밀번호 로그인 항상 가용 |
| NSFW 필터링 API | 이미지 검증 (Stage 1) | 95%+ | 실패 시 수동 리뷰 큐로 전환 |

- VARCO API 버전 변경 시 하위 호환성 유지를 위한 어댑터 레이어를 구축한다
- 모든 외부 API 호출은 타임아웃(30초) + 서킷 브레이커 패턴을 적용한다

### Accessibility

- 비편집기 영역(계정, 프로젝트 목록, 결제, 랜딩)은 WCAG 2.1 AA 준수
- 키보드 네비게이션: 비편집기 영역 전체 지원. 3D 편집기는 Tab 기반 슬라이더 조작 지원
- 색상 대비: 4.5:1 이상 (AA 기준)
- 스크린 리더: 비편집기 영역 ARIA 라벨 적용. 3D 뷰어는 대체 텍스트 설명 제공
- 3D 편집기 자체의 완전한 접근성은 기술적 한계로 MVP에서 제외
